# Walkthrough: OAS-Infra Production Scaffold Setup

The OAS-Infra repository has been successfully transitioned from a conceptual scaffolding setup to a fully deployable infrastructure-as-code repository. We borrowed proven patterns from the `Telephony-Infrastructure` and standalone `Jenkins` reference repos to fill in all the missing gaps.

Here is a summary of the capabilities added:

## 1. Single-Node Kubernetes Provisioning via Terraform
We implemented an EC2 provisioning flow tailored to your target architecture (a single node, rather than a Bastian + Proxy + Server setup like Telephony uses).
- **Files added**: `terraform/main.tf`, `variables.tf`, `security.tf`, `instances.tf`, `outputs.tf`, `templates/userdata.sh.tpl`.
- **Details**: Provisions a `t3.xlarge` instance (Ubuntu 22.04 LTS) with 150GB GP3 storage on a default VPC. The security group limits access to SSH (port 22) and the Kong NodePort (port 30080). It automatically outputs an `env.sh` and `hosts_k8s.yaml` specifically formatted for Kubespray.

## 2. Automated Kubespray Bootstrap
We added automation scripts to deploy Kubernetes directly onto the provisioned node without manual configuration.
- **Files added**: `kubespray/deploy_kubespray.sh`, `kubespray/post_install.sh`.
- **Details**: `deploy_kubespray.sh` automatically copies the inventory file and runs the Ansible playbook using the SSH key generated by Terraform. `post_install.sh` establishes the `platform`, `data`, and `app` namespaces, installs `local-path-provisioner`, sets up ECR credentials, and pulls the `kubeconfig` down to your local machine as `scratch_kubeconfig`.

## 3. Jenkins CI/CD via Docker & Configuration-as-Code
We added a fully reproducible, Dockerized Jenkins server.
- **Files added**: `jenkins/Dockerfile`, `jenkins/docker-compose.yml`, `jenkins/casc.yaml`, `jenkins/plugins.txt`.
- **Details**: The Dockerfile builds an image containing Docker CLI, AWS CLI v2, kubectl, and helm. The Jenkins instance is completely configured via `casc.yaml` (JCasC), which automatically populates AWS/GitHub/SSH credentials based on your `.env` file, and preloads the Job DSL pipeline targeting the `agri-catalogue-service` repository.

## 4. Jenkins EC2 Provisioning (Optional)
If you decide you want Jenkins to run on its own isolated EC2 instead of on a local machine, we provided a ready-made Terraform module for it.
- **Files added**: `jenkins/terraform/*`
- **Details**: Provisions a `t3.medium` instance and bootstraps Docker and Docker Compose, ready to run the aforementioned Jenkins stack.

## 5. End-to-End Orchestrator Script
To ensure these components can be brought up seamlessly without remembering individual commands, we created a single entrypoint script.
- **File added**: `scripts/setup-cluster.sh`
- **Details**: This script runs `terraform apply`, deploys Kubespray, runs post-install configurations, deploys Postgres, Redis, and Elasticsearch (via Bitnami Helm charts), and finally deploys the Kong API gateway.

## 6. Environment Templates and Documentation
- We added an `.env.example` file that specifies all the necessary environment variables for Jenkins and AWS authentication.
- The `README.md` was rewritten. The "Repo Layout" and "Quick Start" sections now clearly outline how to execute the new automation tools sequentially.

## How to proceed
1. Copy `.env.example` to `.env` and fill in your AWS keys, Jenkins passwords, GitHub token, and SSH keys.
2. Run `./scripts/setup-cluster.sh`.
3. Start Jenkins via `docker-compose up -d --build` (in the `jenkins` dir) or use the Jenkins Terraform module.
4. Verify the deployments in the K8s cluster by checking the pods and using Kong to access the service APIs.
